# Phase 2F: deterministic MQL5 nullable runtime

This phase adds a Python-owned source generator only. It does not add a Rust task, AST translation, indicator code, EA code, order placement, or Strategy Tester integration.

## Ownership and command

The implementation is `lab/mql5gen`. The deterministic CLI is:

```bash
PYTHONPATH=. python -m lab.mql5gen --output-dir tests/fixtures/phase2f_mql5_runtime
```

The callable surface is `lab.mql5gen.generate(output_dir, runtime_version=...)`. The destination directory must already exist; generated targets must not already exist.

## Frozen supported inventory

The runtime supports exactly these Phase 2B AST nodes, in this order:

```text
numeric_series, number, boolean_series, compare, and, or, not
```

It supports exactly these operators, in this order:

```text
gt, gte, lt, lte, and, or, not
```

These are the inventory in `engine/labengine/src/ast.rs` and `docs/phase2b_ast_checkpoint.md`; no future grammar is claimed.

## Nullable types and operations

The generated `NoraPhase2RuntimeV1.mqh` defines:

```mql5
enum NoraTriBoolV1
{
   NORA_BOOL_NULL_V1  = -1,
   NORA_BOOL_FALSE_V1 = 0,
   NORA_BOOL_TRUE_V1  = 1
};

struct NoraNullableDoubleV1
{
   bool   is_null;
   double value;
};
```

Numeric null is represented only by `is_null`; no NaN or sentinel value is used. Boolean null is the explicit enum value `-1` and is never converted to false. Constructors and accessors are provided for both types.

## Exact Boolean semantics

Truth-table order for both operands is `null, false, true`.

```text
NOT:  null -> null, false -> true, true -> false

AND:
             null  false true
       null  null  false null
       false false false false
       true  null  false true

OR:
             null  false true
       null  null  null  true
       false null  false true
       true  true  true  true
```

These are the complete strong-Kleene tables implemented by the Rust evaluator (`and`, `or`, and `not`). Evaluation order is left-to-right and deterministic.

## Numeric comparisons

`gt`, `gte`, `lt`, and `lte` compare finite non-null doubles with the direct MQL5 operators `>`, `>=`, `<`, and `<=`. If either operand is null, the result is `NORA_BOOL_NULL_V1`. There is no epsilon and no null-to-zero conversion.

The committed semantic fixture covers each operator with non-null true/false, null-left, null-right, and both-null cases.

## Trigger conversion

`NoraConditionTriggersV1` is the sole condition-to-executable-signal boundary:

```text
true  -> true
false -> false
null  -> false
```

Null remains nullable throughout AST evaluation and is erased only at this final trigger boundary, matching the accepted Rust condition/intents pipeline.

## Semantic fixture and Rust cross-check

`tests/fixtures/phase2f_mql5_nullable_semantics.json` freezes the enum values, complete Boolean tables, all four comparison cases, and trigger conversion. `tests/test_phase2f_mql5.py::test_rust_evaluator_matches_nullable_runtime_fixture` executes the same cases through the real Rust `evaluate_ast` CLI and compares the resulting nullable Parquet values with that fixture. The cross-check passed for NOT, all 9 AND cases, all 9 OR cases, and all 5 cases for each comparison operator.

## Repeatability and failure behavior

Two fresh output directories generated byte-identical headers, equal source hashes, equal runtime identities, and equal manifests. A test-only source mutation and a test-only operator-inventory mutation each changed the runtime identity; neither mutation is committed as a supported runtime.

Generation writes task-owned partial files and validates all inputs before publication. It rejects a missing/non-directory destination, pre-existing generated targets, and an unknown runtime version. Each failure returns exit code 2 through the module CLI, emits no success manifest, and leaves no partial or newly published header. Existing incompatible targets are never overwritten.
